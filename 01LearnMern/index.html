1. Forms and Inputs

HTML

Definition

A form is an HTML element that collects user input and sends it to a server.

Form ek tarah ka container hota hai jisme user apna data bharta hai — jaise login page pe email aur password dalna. <form> tag se banate hain, aur usme <input>, <textarea>, <select> jaisi cheezein hoti hain.

Common Input Types

text email password number checkbox radio file submit date range

Har input ka ek type hota hai — jaise email ke liye type="email" likhte hain, toh browser automatically thoda validation bhi karta hai.

2. Validation Basics

Client-side Server-side

Definition

Validation is the process of checking that user input meets required rules before processing it.

Validation matlab — user ne jo data bhara hai, woh sahi hai ya nahi, ye check karna. Jaise koi email field pe sirf "abc" likhde, toh browser ko batana padta hai ki "bhai, sahi email likho!"

Do types hote hain

Client-side

Browser mein hota hai — user ko turant error dikhta hai. HTML attributes jaise required, minlength, pattern use hote hain.

Server-side

Data server pe jaane ke baad check hota hai. Yeh zyada secure hoti hai — sirf client-side pe depend nahi karna chahiye.

4. Accessibility (a11y)

a11y

Definition

Accessibility means designing websites so that everyone — including people with disabilities — can use them effectively.

Accessibility matlab apni website ko sab ke liye useable banana — jinhe aankhon se dekhne mein takleef ho, keyboard se chalate ho, ya screen reader use karte ho. Yeh sirf "achi baat" nahi — kaafi jagah yeh legal requirement bhi hai.

Key practices

alt text

Images pe alt attribute likho — screen reader padh ke sunaega.

<label>

Har input ke saath ek <label> honi chahiye — bina label ke form confusing hota hai.

keyboard nav

Poori site sirf keyboard se navigate ho sake — Tab, Enter, Arrow keys use karke.

color contrast

Text aur background ke beech contrast zyada hona chahiye — WCAG standard: minimum 4.5:1 ratio.

ARIA roles

aria-label, role, aria-describedby — yeh screen readers ko extra context deta hai.

2. Validation — Advanced

Real-timeRegex

Advanced concept — custom validation + regex pattern

HTML ka built-in required aur type="email" basic check karta hai. Advanced mein hum pattern attribute se custom regex lagate hain, aur JavaScript se real-time feedback dete hain. Password strength bhi JS se calculate hoti hai.

const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/; const phoneRegex = /^[6-9]\d{9}$/; function validateEmail(val) { return emailRegex.test(val); } function validatePhone(val) { return phoneRegex.test(val); } function passwordStrength(pwd) { let score = 0; if (pwd.length >= 8) score++; if (/[A-Z]/.test(pwd)) score++; if (/[0-9]/.test(pwd)) score++; if (/[^A-Za-z0-9]/.test(pwd)) score++; return score; }

1. Box Model Fundamental

Definition

Every element is a rectangular box made of content, padding, border, and margin.

Har HTML element ek box hota hai — andar se bahar tak 4 layers: content (asli data), padding (content ke around space), border (boundary line), aur margin (outside space, doosre elements se gap).

margin
border
padding
content

box-sizing — interview favourite

/* default — width sirf content ka hota hai padding + border alag se add hoti hai */ box-sizing: content-box; /* width mein padding + border bhi included — total size predictable rehta hai */ box-sizing: border-box;

Pro tip Real projects mein hamesha * { box-sizing: border-box; } use karte hain — taaki padding/border add karne se layout na tute.

2. Selectors & Specificity Cascade

Definition

Specificity decides which CSS rule "wins" when multiple rules target the same element.

Jab same element pe ek se zyada CSS rules apply ho rahe hote hain, browser decide karta hai konsa rule jeetega — based on selector ki "specificity score". Higher score wala rule apply hota hai.

Specificity hierarchy — kam se zyada powerful

element / ::pseudo-elemente.g. p, div — lowest priority
.class / [attr] / :pseudo-classe.g. .btn, :hover
#idID selector — class se zyada powerful
inline stylee.g. style="color:red" — sabse zyada
!importantSab kuch override karta hai — sparingly use karo

Interview tip !important ko "escape hatch" samjho — agar har jagah use karo, toh debugging nightmare ban jata hai.

3. Display & Position Layout

Definition

display controls how an element is rendered as a box; position controls where it sits relative to other elements.

Display batata hai element "kaisa" box hai — line mein hai ya apni line pe. Position batata hai woh "kahan" hai — normal flow mein ya specific jagah pe.

display values

blockFull width leta hai, apni line — <div>, <p>
inlineSirf jitni jagah chahiye, line mein — <span>, <a>
inline-blockInline jaisa flow, par width/height set kar sakte ho
noneElement hide — DOM se bhi gayab, jagah nahi leta
flex / gridChildren ko layout karne ka modern tarika

position values

staticDefault — normal document flow mein
relativeApni normal jagah se shift hota hai, jagah reserve rehti hai
absoluteNearest positioned parent ke relative — flow se bahar
fixedViewport ke relative — scroll karne pe bhi waheen rehta hai
stickyScroll tak normal, ek point pe "stick" ho jata hai

Interview classic "visibility: hidden vs display: none?" — hidden wala jagah reserve karta hai (invisible but space leta hai), display:none jagah bhi nahi leta.

4. Flexbox vs Grid Modern layout

Definition

Flexbox arranges items in a single direction (row or column); Grid arranges items in rows and columns simultaneously.

Flexbox 1D hai — ek time pe ya toh row ya column. Grid 2D hai — rows aur columns dono ek saath control karta hai. Navbar, button group → flexbox. Pura page layout, photo gallery → grid.

/* Flexbox — items ek row mein, equal space */ .container { display: flex; justify-content: space-between; align-items: center; gap: 16px; } /* Grid — 3 equal columns */ .container { display: grid; grid-template-columns: repeat(3, 1fr); gap: 16px; }

5. CSS Units px, em, rem, %

Definition

Units determine how a size value is calculated — fixed (absolute) or relative to something else.

pxFixed pixel — kabhi change nahi hota
%Parent element ke size ka percentage
emParent ke font-size ka multiple (nested ho toh compound hota hai)
remRoot (<html>) ke font-size ka multiple — consistent
vh / vwViewport height/width ka percentage

Interview classic "em vs rem?" — em parent pe depend karta hai (nesting mein unpredictable), rem hamesha root pe based hai (predictable). Isliye font sizing mein rem prefer kiya jata hai.

6. Pseudo-classes & Pseudo-elements :hover, ::before

Definition

Pseudo-class (single colon) targets an element's state; pseudo-element (double colon) targets a sub-part that doesn't exist in the DOM.

Pseudo-class — element ki state ya position based pe style — jaise hover ho rahe ho, ya nth child hai. Pseudo-element — naya "virtual" content/part create karta hai jo HTML mein likha nahi hai.

/* Pseudo-classes — single colon */ a:hover { color: blue; } li:nth-child(2) { font-weight: bold; } input:focus { border-color: green; } input:disabled { opacity: 0.5; } /* Pseudo-elements — double colon */ p::first-line { font-weight: bold; } .box::before { content: "→ "; } .box::after { content: ""; display: block; }
Top Interview Questions — click to expand Q&A
1. Difference between em and rem?

em apne immediate parent ke font-size se calculate hota hai — nested elements mein compound (multiply) ho jata hai. rem hamesha <html> root element ke font-size se calculate hota hai — isliye predictable rehta hai.

2. visibility: hidden vs display: none?

display: none element ko completely render tree se hata deta hai — koi space nahi leta, screen reader bhi skip karta hai. visibility: hidden element ko invisible kar deta hai par uski jagah (layout space) reserve rehti hai.

3. What is the CSS Box Model?

Har element ek box hota hai jisme content, padding, border, aur margin layers hoti hain — andar se bahar. box-sizing: border-box se width calculation predictable banayi ja sakti hai.

4. position: relative vs absolute?

relative element apni original jagah se offset hota hai, par uski jagah document flow mein reserve rehti hai. absolute element document flow se completely bahar nikal jata hai aur nearest positioned ancestor (jiska position static na ho) ke relative position hota hai.

5. What is specificity, and how is it calculated?

Specificity decide karti hai konsa CSS rule apply hoga jab multiple rules clash karein. Order (low to high): element selectors → class/attribute/pseudo-class → ID → inline styles → !important. Zyada specific selector jeet jaata hai.

6. Flexbox vs Grid — kab kya use karein?

Flexbox 1-dimensional layout ke liye best hai (single row/column) — navbars, button groups, centering content. Grid 2-dimensional layout ke liye — full page structure, image galleries, dashboards jahan rows aur columns dono control karna ho.

7. What is a stacking context and z-index?

z-index decide karta hai kaunsa element upar dikhega jab elements overlap karte hain — higher value upar. Yeh sirf positioned elements (relative, absolute, fixed, sticky) pe kaam karta hai. Stacking context naya tab banta hai jab koi element position + z-index, ya opacity < 1, ya transform jaisi properties use karta hai — uske andar ke z-index sirf usi context mein compare hote hain.

8. What is the difference between inline, inline-block, and block?

block apni full line leta hai aur width/height set ho sakti hai. inline sirf content jitni jagah leta hai, width/height set nahi hoti, aur line mein doosre elements ke saath flow karta hai. inline-block dono ka mix — line mein flow karta hai par width/height set kar sakte ho.

9. What is the CSS cascade, and how does it work with inheritance?

Cascade woh algorithm hai jisse browser decide karta hai conflicting styles mein kaunsa apply hoga — origin (browser/user/author), specificity, aur source order (later rule jeet jaata hai agar specificity same ho) ke basis pe. Inheritance alag concept hai — kuch properties (jaise color, font-family) child elements ko parent se automatically mil jaati hain, jab tak override na kiya jaye.

10. What is a media query, and why is it important?

@media query se hum screen size, orientation, ya device type ke basis pe different CSS apply kar sakte hain — responsive design ka foundation. e.g. @media (max-width: 768px) { ... } — mobile screens ke liye alag styles.
